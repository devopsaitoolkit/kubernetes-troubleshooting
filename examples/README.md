# Examples

Realistic, **synthetic** manifests that reproduce common failure modes in a
disposable test cluster (kind / minikube / k3d). Use them to practice the
diagnostic flows in `docs/errors/` safely — never apply them to production.

Each example links back to the error page it demonstrates.

> ⚠️ These manifests intentionally misconfigure things (bad images, impossible
> resource requests, broken probes). Apply only in a throwaway cluster.
